What is AI DevKit?
An operating model for AI-centric software engineering
  • AI writes specs, tests, code, and docs — humans approve the spec and the release
  • Works with your existing stack: git, repos, PRs, continuous integration
  • Four pillars: spec planning, test-driven implementation, progressive disclosure docs, dev environment
  • Plain markdown — works with any agent that can read repo files
The Lifecycle
Lead AI executes the workflow — Verify AI independently reviews all work
Plan
Decide in the spec, not in the code
  • Lead AI reads requirement, docs and code — drafts spec with acceptance criteria and design decisions
  • Verify AI independently reviews the spec — mismatches reveal ambiguity before any code is written
  • Decisions made in the spec, not deferred to implementation
  • Human signs off the spec before work begins
Implementation
Test-driven development with AI separation
  • Red-Green-Refactor: Verify AI authors the tests from the spec, Lead AI implements
  • Independent test authorship prevents AI from testing its own assumptions
  • After TDD: update Progressive Disclosure docs, Verify AI cross-checks against code
  • Specs archived — docs carry everything forward
Progressive Disclosure Docs
The same 8 files in every repo — any agent can pick up any repo
  • Level 0: Identity card — what is this repo?
  • Level 1: Eight fixed files — setup, architecture, code map, conventions, workflows, interfaces, gotchas, security
  • Level 2: On-demand deep dives when Level 1 isn’t enough
  • Levels 0 + 1 load upfront, Level 2 only when needed — full context without filling the context window
AI Dev Environment
One workspace where AI can run the whole system end to end
  • Container of containers — each component isolated, all accessible, runs locally or in the cloud
  • Cloud workspaces for team handoff — another engineer picks up where you left off
  • Built-in tooling: Playwright for browser testing, Terraform for infrastructure
  • Session audit trail — reproducibility, traceability, and evals for debugging agent behaviour
The Prompts
Six markdown files — curl from GitHub, pipe to any agent
  • Six prompts: spec, implement, create-docs, update-docs, verify-codex, verify-claude
  • Chain work + verify in either direction, straight from the repo:
    curl -sL
      https://raw.githubusercontent.com/AgoraIO-Community/ai-devkit/main/prompts/create-docs.md
      https://raw.githubusercontent.com/AgoraIO-Community/ai-devkit/main/prompts/verify-codex.md
      | claude --dangerously-skip-permissions
  • ~50 min end to end, zero human intervention — full progressive disclosure docs for any repo
  • Plain text, not hidden skills — but skills and slash commands can be hooked up to the prompt files depending on implementation preferences
Getting Started
Pick one repo, run one command
  • Pick a repo — use the create-docs prompt chained with verify as shown in the readme
  • Let it run — now you have Progressive Disclosure docs enabling any agent to work effectively in that repo
  • Add specs and test-driven implementation using the relevant prompts
  • Build your AI Dev Environment to run multi-repo systems end to end
